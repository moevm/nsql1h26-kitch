## Тема курсовой работы 

**34 Информационная система компании по производству и монтажу кухонный гарнитуров**

**Сложность:** Сложная 

**Предлагаемая СУБД:** MongoDB

**Набор данных:** Синтетические данные

**Комментарий:** Задача - сделать сервис для бизнеса, который занимается кухонными гарнитурами. Основной акцент - прототип системы управления производством (заказы, раскрой, рабочие часы, логистика)

## Запуск

1. **Установить зависимости**
```bash
pip install -r requirements.txt
```

2. **Запустить базу данных в Docker-compose**
```bash
docker-compose up -d
```

3. **Запустить основной скрипт**
```bash
python main.py
```

## Роли в приложении

### Покупатель

- Создаёт заказ, указав нужные параметры

- Отслеживает заказ

### Рабочий

- Получение задачи и выполнение её

- Может менять статус заказ после получения задачи

- Может просматривать детали заказа

### Админ (владелец бизнеса)

- Может просматривать всё

- Просмотр финансовой аналитики


## Коллекции

### Users

- id
- name
- role (покупатель, рабочий, админ)
- email
- phoneNumber
- password
- createAt (когда создан аккаунт)
- defaultAddress (адрес по-умолчанию)

### Orders

- id
- userId
- description (комментарий)
- status (в обработке, в работе и т.п.)
- createAt (время создания заказа)
- updateAt (время обновления заказа)
- deadline (крайний срок выполнения заказа)
- deliverTo (адрес доставки)

### Tasks

- id
- orderId
- workerId (id исполнителя (рабочего))
- deadline (крайний срок выполнения заказа)

### Сutting (Раскрой)

- id
- materialId
- blueprint (чертёж раскроя)
- type (тип изделия)
- createAt
- updateAt


### Materials

- id
- type ()
- name (наименование (дуб 300х3000х30 мм))
- remain (сколько в наличии)
- cost (стоимость)
- updateAt (время последнего обновления)
